Vue中使用CSS Modules

发表于 2020-04-20 | 分类于 WEB开发
字数统计: 885 | 阅读时长 ≈ 3

CSS Modules是一个流行的，用于模块化和组合 CSS 的系统。vue-loader 提供了与 CSS Modules 的一流集成，可以作为模拟 scoped CSS 的替代方案。

本文将详细介绍如何在Vue中使用css modules。

与scoped对比

最开始使用Vue的时候，提倡并大量使用的是scoped技术

这个可选 scoped 属性会自动添加一个唯一的属性 (比如 data-v-21e5b78) 为组件内 CSS 指定作用域，编译的时候 .list-container:hover 会被编译成类似 .list-container[data-v-21e5b78]:hover。但是，它并不能完全避免冲突。

以上面的代码为例，使用scoped之后，它在元素上添加了一个唯一的属性’data-v-0467f817’，CSS样式被编译如下:

但是，如果用户也定义了一个errShow类名，会影响到所有定义为errShow类名的组件的显示。
而CSS modules则做的更彻底，它不是添加属性，而是直接改变类名。

这样，大大降低了冲突的可能性，只要不是用户直接给span标签设置样式，基本上不会影响组件的显示。
